### 4. Edição de perfil + upload de foto
- `Perfil.jsx` reescrito com:
  - Avatar clicável com ícone de câmera para trocar foto
  - Nome editável inline (ícone de lápis → campo de texto → ✓ / ✗)
  - Feedback de erro exibido na tela
- Novas funções em `src/lib/api.js`:
  - `updateProfile(userId, { nome, foto_url })` — atualiza dados do perfil
  - `uploadAvatar(userId, file)` — faz upload para Supabase Storage bucket `avatars`, retorna URL com cache-busting
- Bucket `avatars` criado no Supabase Storage (público)
- Policies de storage aplicadas via SQL:
  ```sql
  create policy "Avatar insert" on storage.objects for insert to authenticated with check (bucket_id = 'avatars');
  create policy "Avatar update" on storage.objects for update to authenticated using (bucket_id = 'avatars');
  create policy "Avatar select" on storage.objects for select using (bucket_id = 'avatars');
  ```
- `object-cover` → `object-contain` em todos os avatares de jogadores (evita cortar a imagem)

### 5. Correção do nome no cadastro
- **Problema:** ao registrar com email/senha, o nome ficava com o email (trigger criava o perfil antes do `update`)
- **Solução:** passar `full_name` nos metadados do `signUp`:
  ```js
  supabase.auth.signUp({ email, password, options: { data: { full_name: nome } } })
  ```
  Assim o trigger já lê `raw_user_meta_data->>'full_name'` e cria com o nome correto.
